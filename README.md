# Проект: Сортировщик мусора (Eco-Agent / VLM + OpenCV)

Система разделена на **Серверную часть** (обработка нейросетью и веб-дашборд) и **Клиентскую часть** (камера, захват изображений). Это позволяет снизить нагрузку на слабые устройства (например, Raspberry Pi), перенеся тяжелые вычисления на более мощный сервер.

## Структура проекта
- `server.py` — веб-сервер, который принимает фотографии от камеры, запускает анализ и показывает историю сканирований.
- `only_camera.py` — легкий клиент, который только делает фото с камеры и отправляет их на сервер.
- `run_scan_binary.py` — скрипт, используемый сервером для обработки фотографий в оперативной памяти.
- `run_scan.py` — утилита для тестирования распознавания на локальных файлах (без сервера).
- `models.py` / `schemas.py` / `camera.py` — модули для работы с камерой, нейросетью и структурами данных.
- `setup_pi.sh` — скрипт автоматической настройки Raspberry Pi.

## 🚀 ЗАПУСК СИСТЕМЫ

Система состоит из трёх основных компонентов, каждый из которых необходимо запустить в отдельном терминале (или на разных устройствах):

### 1. Сервер (AI Backend + API)
Сервер принимает фотографии, обрабатывает их нейросетью и отдает JSON-данные.
```bash
# В терминале 1
python3 server.py
# Запустится на порту 8060
```

### 2. Веб-интерфейс (Frontend WebApp)
Современный интерфейс пользователя (Eco Reward) для отображения результатов и истории.
```bash
# В терминале 2
cd webapp/vite-app
npm run dev
# Запустится локальный dev-сервер (по умолчанию на порту 5173)
```

### 3. Камера (Разведывательный Клиент на Raspberry Pi / ПК)
Тонкий клиент, который делает фото по нажатию `Enter` и отправляет его серверу (шаг 1).
```bash
# В терминале 3 
python3 only_camera.py
# Если API-сервер запущен на другом устройстве:
# python3 only_camera.py http://<IP_сервера>:8060/api/upload
```

## Как работает логика
1. **Клиент (`only_camera.py`)**: По нажатию ENTER захватывает кадр через OpenCV, переводит его в байты (`image_bytes`) и делает HTTP POST запрос на сервер.
2. **Сервер (`server.py`)**: Принимает POST-запрос, открывает новый независимый поток и передает байты картинки напрямую в модуль `run_scan_binary.py`.
3. **AI Анализ (`run_scan_binary.py`)**: Внутри фонового потока байты передаются квантованной модели машинного зрения, которая производит классификацию и описание объекта (строгий JSON формат).
4. **Хранение**: Картинка сохраняется в папку `assets/`, а результат (JSON) в `scans/`.
5. **Дашборд**: Пользователь открывает `http://localhost:8060`, где видит все сканы (авто-обновление каждые 3 секунды).

## Оптимизация для 2GB RAM (Raspberry Pi)
- Прямая работа с бинарными данными в памяти, чтобы избежать задержек записи/чтения временных файлов.
- Анализ картинок запускается в фоновом потоке сервера, чтобы не блокировать веб-интерфейс.
- Нейросеть квантована для скорости и приемлемого потребления памяти.
